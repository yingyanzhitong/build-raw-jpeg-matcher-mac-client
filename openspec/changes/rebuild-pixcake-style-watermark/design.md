## Context

项目是 Tauri 2、React 19 与 Rust 构成的单窗口 macOS 图片工具。旧水印前端已经不在 `App` 中挂载，但旧页面、Rust 命令、依赖、规格和截图仍然存在；旧实现同时承担文字渲染、不可验证的去背景和平铺分支，并且预览直接访问任意本地路径、导出不保留 EXIF/ICC，也没有进度与取消。

新版面向本地摄影批处理，必须延续项目现有的工作区标签、系统文件选择器、底部日志、受限临时预览目录和安全导出约束。所有文档、状态与图像处理均留在本机。

## Goals / Non-Goals

**Goals:**

- 建立独立、可测试的图片水印领域模型和全新 Rust 图像处理管线。
- 为横图、竖图、方图提供独立配置和一致的 Canvas/导出几何行为。
- 为图片标志提供不依赖 AI 的本地透明玻璃材质近似，并提供单个/平铺两种布局。
- 保留源目录结构、EXIF、ICC 和正确显示方向，同时绝不覆盖已有文件。
- 通过 IPC Channel 实时报告批量进度，并在当前文件完成后安全取消。
- 提供符合现有 macOS 工具视觉语言的键盘、VoiceOver 和窗口缩放体验。
- 让水印工作区与“图片 / RAW 匹配”使用相同的侧栏节奏、工具栏高度、空状态和表面层级。

**Non-Goals:**

- 不提供文字水印、阴影、模板库或旧配置兼容层。
- 不提供人物、商品或复杂照片背景分割；不透明素材分割仅处理具有近似纯色背景的标志素材。
- 不支持 RAW、TIFF、HEIC、双份导出、云同步、服务端或 AI。
- 不做完整 ICC 色彩空间转换；只保留目标照片原有 ICC 数据。
- 不保留 XMP/IPTC，也不承诺 JPEG 重新编码后像素无损。

## Decisions

### 1. 删除后重建，不复用旧领域模型

旧 `WatermarkView`、类型和 Rust 实现整体删除，再创建 `WatermarkWorkspace`、纯状态函数和新的后端模块。新版平铺和背景透明化使用可测试的新领域模型，不复用旧配置或处理分支。继续在原文件上裁剪的替代方案会保留大量无效分支和旧测试，因此不采用。

### 2. 三画幅使用同构归一化配置

`WatermarkProfiles` 包含 `landscape`、`portrait`、`square` 三个 `WatermarkProfile`。配置增加 `layout: single | tile` 和基于目标短边的 `tileSpacingPercent`。单个模式下，尺寸是目标短边的百分比；偏移分别是目标宽高百分比；九宫格先给出基准位置，再应用偏移，最后按旋转后外接矩形和 3% 短边安全边距钳制。平铺模式不使用九宫格，以旋转后外接矩形加间距生成覆盖画面的规则网格，偏移移动整个网格相位。Canvas 和 Rust 分别实现相同公式，并使用相同测试数据。

方图只在方向校正后的宽高相等时成立；其余按宽高大小归为横图或竖图。选中预览图片会自动切换对应配置；同步操作用值复制当前配置到另外两类，不共享对象引用。

### 3. 所有素材先生成本地透明玻璃版本

素材解码后若已包含透明像素，则使用原 alpha 作为主体蒙版；若完全不透明，后端通过整图量化直方图估算占比最高的背景主色，按固定容差与羽化区间生成主体蒙版，并裁掉素材最外侧的极窄装饰边框，避免边框被误判为玻璃主体。后端在主体内生成低 alpha 中性填充，在外轮廓和原图内部明显颜色交界处生成较亮高光，并在高光内侧生成克制暗边，得到类似 Liquid Glass 的本地静态材质近似。背景保持完全透明，主体内部允许底图清晰透出；不调用 AI，也不依赖目标照片内容。若素材没有可保留主体则拒绝素材并提示改用有效图片。

玻璃 PNG 预览写入受限临时目录，导出重新运行同一算法，不信任或读取前端 Canvas 像素。该方案不模拟系统级实时折射，只保证透明填充、轮廓高光、内部交界和预览/导出一致性。

### 4. 预览资产经过后端验证并写入受限临时目录

扫描只返回路径和元数据。递归扫描、图片解码和预览缩放通过异步命令转入阻塞线程池，避免占用 Tauri 主线程；当前图片与水印素材应用 EXIF 方向后，以适合屏幕预览的三角插值输出 PNG 到 `$TEMP/raw-jpeg-matcher-thumbnails/**`，前端再通过现有 asset protocol 加载。相较放宽 CSP/asset scope，这种方案不会授权 WebView 任意读取用户目录。缩略图仅对当前窗口化条目按需请求。

### 5. Rust 负责最终像素与元数据管线

后端使用 `image 0.25.10` 读取方向、EXIF 和 ICC，应用方向到像素后移除 EXIF 方向标记。水印先生成玻璃 RGBA 图层，再按短边比例缩放，通过精简后的 `imageproc` 做任意角度旋转，最后按单个或平铺位置进行通透度和 alpha 合成。平铺数量设置上限，避免极小尺寸产生失控循环。JPEG 使用质量 100，PNG 使用无损编码；编码器支持时写回清理后的 EXIF 和原 ICC，写回失败记录警告但不丢弃已处理图片。

目标文件使用 `create_new` 原子创建。后端重新规范化根目录和每个来源路径、拒绝符号链接逃逸、拒绝输出目录位于输入目录内部，并自行计算相对路径，不信任前端提交的相对路径。

### 6. Channel 进度与原子取消

前端用 `crypto.randomUUID()` 生成 `jobId`，调用 `export_watermarked_images(request, onEvent)`。后端全局状态只保存当前任务的 `Arc<AtomicBool>`；同一时间只允许一个水印任务。每张结束发送 `itemFinished`，元数据降级发送 `warning`。`cancel_watermark_export` 只设置标记，循环在当前图片完成后停止，返回包含取消剩余数的汇总并清理任务状态。

### 7. 配置只在浏览器本地持久化最小字段

使用版本化键 `watermark-settings:v3` 保存水印路径和三套配置，加载时可读取 `v2`/`v1`，将旧 `opacity` 数值迁移为 `clarity` 并为缺失布局字段补默认值；所有读写包裹 `try/catch`，加载时逐字段校验并钳制范围。`clarity` 范围为 0–1，1 表示最高通透度，但仍保留最低轮廓可见度。输入目录、输出目录、图片列表、日志和任务状态不持久化。恢复水印路径后必须重新调用素材检查命令并重新生成玻璃版本；失败则清空路径并记录警告。

### 8. 工作区保持常驻并复用统一日志

`App` 增加 `watermark` 标签和状态，三个面板通过 `hidden` 切换而不卸载，因此切换标签不会中断导出。`Cmd/Ctrl+3` 激活水印，方向键、Home、End 在三个标签间循环；水印日志进入现有 `LogBottomSheet`。布局采用 320px 设置检查器和弹性预览区，适配当前 980px 最小窗口。

### 9. 水印工作区对齐匹配工作台的视觉层级

“图片 / RAW 匹配”是当前应用的工作台基准。水印侧栏改为同宽的 312px，并将照片来源、水印素材、画幅配置表达为真实的三步流程：未完成步骤显示编号和“当前步骤/待完成”，完成的来源与素材显示勾选和“已完成”。步骤状态只表达已有业务就绪条件，不新增向导或阻塞配置操作。

右侧使用相同的 64px 顶部结果栏，标题、数量徽标、说明和操作按钮与匹配工作区共享排版节奏。预览区域不再同时使用整面棋盘格、卡片边框和画幅虚线框；空状态直接置于 `empty-workbench` 舞台，真实照片在同一舞台居中。棋盘格仅保留给需要表达透明度的水印素材缩略图。上一张、缩略图和下一张保留为水印特有导航，但改为贴底、带顶部分隔线的工具条，不再形成第二张浮动卡片。

该调整只改变 React 结构和样式，不修改扫描、配置持久化、Canvas 几何、Rust 合成或导出协议。最小窗口下仍退化为单列滚动布局，键盘焦点和已有 ARIA 关系保持不变。

## Risks / Trade-offs

- [Canvas 与 Rust 插值算法可能产生轻微像素差异] → 几何参数共用规范和测试；预览承诺位置/大小一致，不承诺逐像素一致。
- [JPEG 质量 100 仍会重编码且可能增大文件] → 在界面明确提示，源文件始终保留。
- [部分 EXIF/ICC 容器不被编码器接受] → 输出图片继续生成，逐项发送元数据警告并计入日志。
- [大量文件产生过多预览开销] → 图片扫描不生成缩略图，只为当前选择和窗口化缩略图按需缓存。
- [取消发生在单张大图处理中时不会立即响应] → 明确为“当前图片完成后停止”，避免在写文件中途留下损坏目标。
- [纯色背景容差可能漏检低对比主体或保留复杂纹理] → 不透明输入只承诺近似纯色背景标志；界面展示玻璃处理结果，复杂素材建议先制作为透明 PNG。
- [静态 RGBA 无法复刻系统级实时折射] → 本期以透明填充、明暗双边和内部颜色交界近似 Liquid Glass，并让 Canvas 与 Rust 使用同一玻璃素材和通透度公式。
- [极小水印平铺可能产生大量叠加] → 间距设置最小值并限制单张图片的最大平铺数量，前后端使用同一约束。

## Migration Plan

1. 删除旧实现、旧变更和旧截图，同时移除只服务于文字水印的依赖。
2. 建立新后端命令、状态与测试，再接入全新前端工作区。
3. 运行前端状态测试、Rust 测试和生产构建；手工验证三个画幅、方向、元数据和取消。
4. 旧水印从未在当前主界面接入，因此无需迁移用户任务；新版使用新的本地配置键。
