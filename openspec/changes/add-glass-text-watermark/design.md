## Context

当前水印模块已经把图片素材转换为玻璃 RGBA 资产，并让 Canvas 预览和 Rust 导出复用同一份资产、通透度公式、三画幅配置及单个/平铺几何。新增文字水印需要跨越 React 状态、本地配置、Tauri 命令、系统字体发现和 Rust 栅格化，同时不能恢复已删除的旧文字水印处理分支。

本期只面向 macOS 本机字体，文字和字体数据不得发送到网络。现有图片水印配置必须无损迁移。

## Goals / Non-Goals

**Goals:**

- 在同一素材面板切换图片和文字水印，并保留两种素材各自的输入状态。
- 枚举并校验 macOS 本机字体，以稳定字体标识完成预览、恢复和导出。
- 把单行文字栅格化为透明 alpha 蒙版，再复用现有玻璃材质生成函数。
- 让文字大小、旋转、位置、通透度和平铺在 Canvas 与 Rust 导出中一致。
- 让文字默认通透度足够可辨，并以交错行缩短长文本平铺的视觉空档。
- 让长文字旋转后仍按字高形成紧凑平铺，不再由旋转外接框放大行距。
- 让图片与文字在相同通透度数值下具有接近的感知可见度，同时保持同一条滑块曲线。
- 将 v3 设置无损迁移到 v4，再以 v5 分离图片与文字 profile；失效字体安全回退。

**Non-Goals:**

- 多行排版、富文本、字距、行距、描边颜色、渐变、弧形文字或自定义字体文件导入。
- CoreText 特效、真正的实时折射或依赖目标照片内容的动态玻璃着色。
- 云字体、字体下载、AI 字体推荐或任何服务端处理。

## Decisions

### 1. 使用带标签的统一水印来源

公开导出请求把单一 `watermarkPath` 替换为 `source`：`image { path }` 或 `text { text, fontId }`。后端先把任一来源准备为统一的 `PreparedWatermarkAsset`，后续扫描校验、旋转、平铺、合成和编码不分叉。相比增加第二套文字导出命令，这能保证图片与文字共享安全规则和任务语义。

### 2. 本机字体使用 PostScript 名作为稳定标识

后端通过 `fontdb` 一次加载系统字体，返回按显示名排序的 `WatermarkFontInfo`。持久化和 IPC 只传 `postScriptName`，不暴露或信任前端提交的字体文件路径；渲染时重新在字体库中匹配该标识并读取对应 face index。默认字体优先选择 `PingFang SC` 常规体，其次选择 Helvetica/Arial，再回退到首个可用字体。

### 3. 文字先生成高分辨率 alpha 蒙版，再玻璃化

`ab_glyph` 与 `imageproc` 只负责在固定高分辨率下测量和栅格化单行文字。栅格结果是白色 RGB 与抗锯齿 alpha；随后调用现有 `create_glass_watermark`，生成低 alpha 填充、外轮廓高光和内侧暗边。文字限制为去除首尾空白后的 1–120 个字符，不接受换行或控制字符，避免不可控画布和批量导出成本。

### 4. 文字大小复用三画幅 `sizePercent`，但按高度解释

图片模式保持“尺寸（目标短边）”，以玻璃资产宽度占目标短边的百分比缩放；文字模式显示“文字大小（目标短边）”，以文字玻璃资产高度占目标短边的百分比缩放。`WatermarkAssetInfo` 增加 `sizeBasis: width | height`，Canvas 几何与 Rust 图层准备都读取相同基准。这样字号随目标分辨率稳定，长文本只增加宽度，不会被强制压成同一宽度。

### 5. 文字设置采用 v4 本地存储并延迟生成预览

`watermark-settings:v4` 保存 `sourceKind`、图片路径、文字内容、字体标识和三套 profile；v3/v2/v1 加载时默认迁移为图片模式。文字或字体变化后前端以短延迟合并调用 `inspect_text_watermark`，避免连续输入触发无效栅格化；过期响应通过请求序号丢弃。字体目录加载失败只禁用文字模式，不影响图片水印。

### 6. 文字默认参数与平铺排列独立于图片水印

设置升级为 `watermark-settings:v5`，分别保存图片和文字的三画幅 profile。文字 profile 默认通透度为 40%、平铺间距为目标短边的 2%；图片 profile 继续使用既有默认值。v4 迁移时，只有仍等于旧默认值的文字参数才替换为新默认值，用户已经调整过的通透度和间距保持不变。

文字平铺按行交错半个横向步长。单个文字外接框与间距仍不重叠，但相邻行填入上一行的横向空档，避免长文本整列对齐后产生接近两倍短边的视觉间隔。Canvas 与 Rust 使用相同的整数步长和奇偶行规则；图片平铺仍保持原有对齐方式。

### 7. 在文字玻璃素材层校准感知可见度

图片与文字继续共享 `clarity` 到 alpha 系数的同一条曲线，不为文字引入隐藏的第二套滑块换算。文字栅格化后生成的玻璃素材由于笔画细、抗锯齿像素占比高，在相同系数下视觉重量更弱，因此仅在文字玻璃素材生成完成后把非零 alpha 乘以 1.35，并把最大 alpha 限制为 240，增强填充与暗边但不产生完全不透明像素。

Canvas 预览与 Rust 导出继续读取同一份已校准 RGBA 素材，因此无需在前端叠加额外补偿，也不会出现预览与导出两套公式。文字预览缓存版本递增，确保升级后不会继续复用旧的弱可见度缓存；图片玻璃素材及缓存保持不变。

### 8. 文字平铺使用随文字旋转的局部网格

旧平铺先旋转每个文字素材，再用旋转外接矩形宽高作为横纵步长。长文字在有角度时会把宽度投影到外接矩形高度，使纵向步长远大于“字高 + 间距”，因此把间距降到 1% 仍然稀疏。

文字平铺改为在文字自身局部坐标系中建立网格：横向步长为未旋转字宽加间距，纵向步长为未旋转字高加间距，奇偶行在局部横轴错开半个横向步长；随后把所有网格中心与文字使用相同角度旋转到目标图片坐标。这样局部矩形保持不重叠，同时旋转角度不会额外放大行距。横纵偏移继续作用于整个网格，Canvas 与 Rust 对网格中心和图层左上角使用相同舍入规则。

为了覆盖目标画面，系统按目标对角线、偏移量和旋转图层外接框计算局部行列搜索半径，只保留与画面相交的实例并继续执行单张 5,000 个实例的安全上限。图片水印保留既有按旋转外接框对齐的网格，避免改变已配置的图片效果。

## Risks / Trade-offs

- [部分字体不包含用户输入字符] → 栅格化后若没有有效字形则拒绝，并提示更换字体；本期不实现逐字符字体回退。
- [系统字体数量较多，首次枚举可能稍慢] → 后端进程内缓存字体数据库和排序结果，前端只请求一次。
- [PostScript 名可能在系统升级后消失] → 启动时重新校验并回退默认字体，同时写入警告日志。
- [文字玻璃是静态 RGBA 近似] → 复用已验收的透明填充、明暗双边和通透度公式，确保预览与导出一致。
- [文字校准过强会失去玻璃感] → 只增强非零 alpha，并以 240 为上限，任何通透度下都保留透明度。
- [长文字按高度缩放后可能超过画面宽度] → 单个布局继续按安全边距整体缩小；平铺在导出前应用数量上限校验。
- [紧凑网格可能增加平铺实例数量] → 只保留与画面相交的实例，并在预览与导出中维持 5,000 个实例上限。

## Migration Plan

1. 新增 v5 配置读取，优先读取 v5，找不到时按 v4 → v3 → v2 → v1 迁移。
2. v4 的共享 profile 同时作为图片 profile 基础；文字 profile 仅把仍为旧默认的 100% 通透度和 8% 间距迁移为 40% 与 2%。
3. 发布后首次保存写入 v5，不删除旧键，便于回滚到旧版本。
4. 回滚时旧版本继续读取 v4 或 v3 配置；v5 的来源独立 profile 会被忽略，不影响旧版本启动。
