## ADDED Requirements

### Requirement: 图片输入支持 JPG、JPEG 和 PNG
系统 MUST在“图片 → RAW”方向允许用户通过文件选择、目录选择、拖放或文本清单加入图片输入，并将 .jpg、.jpeg 和 .png 作为受支持的图片扩展名；扩展名判断不区分大小写。

#### Scenario: 选择多种图片文件
- **WHEN** 用户选择 IMG_1001.JPG、IMG_1002.jpeg 和 IMG_1003.PNG
- **THEN** 系统将三个文件都加入当前方向的图片输入

#### Scenario: 递归扫描图片输入目录
- **WHEN** 用户选择或拖入包含多层子目录的图片目录
- **THEN** 系统递归加入其中所有受支持的 JPG、JPEG 和 PNG 文件

#### Scenario: 混合拖入受支持和不支持的文件
- **WHEN** 用户拖入的内容同时包含受支持图片和其他文件
- **THEN** 系统加入有效图片，跳过不支持的文件，并在当前方向日志中汇总已跳过数量

#### Scenario: 通过文本清单加入图片引用
- **WHEN** 用户在“图片 → RAW”方向输入每行一条的图片文件名、路径或文件名后缀
- **THEN** 系统将去除图片扩展名后的内容作为手工输入引用加入当前任务

### Requirement: RAW 输入支持文件、目录、文本清单和格式筛选
系统 MUST在“RAW → 图片”方向允许用户通过文件选择、目录选择、拖放或文本清单加入 RAW 输入。物理文件仅接受当前启用的受支持 RAW 格式；文本清单可填写 RAW 文件名、路径或末尾编号，带扩展名时同样仅接受当前启用的受支持 RAW 格式。

#### Scenario: 选择多个 RAW 文件
- **WHEN** 用户在“RAW → 图片”方向选择多个当前已启用格式的 RAW 文件
- **THEN** 系统将所有有效 RAW 文件加入当前方向输入

#### Scenario: 递归扫描 RAW 输入目录
- **WHEN** 用户选择或拖入包含嵌套 RAW 文件的目录
- **THEN** 系统递归扫描目录，并只加入当前已启用格式且通过 RAW 文件校验的文件

#### Scenario: 通过文本清单加入 RAW 引用
- **WHEN** 用户在“RAW → 图片”方向输入每行一条 RAW 文件名、路径或文件名后缀
- **THEN** 系统将去除 RAW 扩展名后的内容作为手工输入引用加入当前任务；带未启用或不支持 RAW 扩展名的条目必须跳过并记录

### Requirement: RAW 格式列表必须统一
系统 MUST将 .cr2、.cr3、.nef、.arw、.raf、.orf、.rw2、.dng、.rwl、.pef、.3fr 和 .iiq 作为可选择的 RAW 格式，并在前端与后端使用同一份格式定义；扩展名判断不区分大小写。

#### Scenario: 展示完整 RAW 格式
- **WHEN** 用户查看任一方向中的 RAW 格式筛选
- **THEN** 系统展示全部十二种受支持 RAW 格式及当前选中状态

#### Scenario: RAW 格式变化影响现有 RAW 输入
- **WHEN** 用户在“RAW → 图片”方向取消某个 RAW 格式
- **THEN** 系统从当前有效输入集合中排除该格式的文件、记录变化并使当前匹配结果失效

### Requirement: 输入列表按规范化路径去重
系统 MUST在每个方向内按规范化后的物理文件路径去重，不得因同一文件通过文件选择、目录扫描或拖放被多次发现而生成重复输入；不同路径但主文件名相同的文件仍视为独立输入。

#### Scenario: 重复加入同一路径
- **WHEN** 同一个文件通过多个入口被重复加入当前方向
- **THEN** 系统仅保留一条输入，并在当前方向日志中记录重复项

#### Scenario: 不同路径具有相同主文件名
- **WHEN** 两个不同目录中的文件具有相同主文件名
- **THEN** 系统将它们保留为两个独立输入

### Requirement: 每个方向使用一个查找目录
系统 MUST为当前方向保存且仅保存一个候选查找根目录：“图片 → RAW”使用 RAW 查找目录，“RAW → 图片”使用图片查找目录；目录扫描必须包含其子目录。

#### Scenario: 选择 RAW 查找目录
- **WHEN** 用户在“图片 → RAW”方向选择一个目录
- **THEN** 系统将其保存为当前方向的 RAW 查找根目录

#### Scenario: 选择图片查找目录
- **WHEN** 用户在“RAW → 图片”方向选择一个目录
- **THEN** 系统将其保存为当前方向的图片查找根目录

#### Scenario: 取消目录选择
- **WHEN** 用户打开目录选择器后取消操作
- **THEN** 系统保留当前方向原有的查找目录

### Requirement: 拖放必须按当前方向路由
系统 MUST根据当前方向解释拖放路径，不得始终将拖入内容按图片输入处理。

#### Scenario: 在 RAW 找图片方向拖入文件
- **WHEN** 当前方向为“RAW → 图片”且用户拖入文件或目录
- **THEN** 系统按 RAW 输入规则收集内容

#### Scenario: 在图片找 RAW 方向拖入文件
- **WHEN** 当前方向为“图片 → RAW”且用户拖入文件或目录
- **THEN** 系统按图片输入规则收集内容

### Requirement: 两步条件就绪后自动执行匹配
系统 MUST仅在当前方向至少存在一个有效输入、已选择查找目录、RAW 格式集合非空且没有运行中任务时执行匹配；当文件输入或查找目录的最后一个缺失条件被补齐时，系统必须自动开始对应方向的匹配。

#### Scenario: 条件完整
- **WHEN** 当前方向的输入、查找目录和 RAW 格式均有效且系统空闲，并且用户刚完成输入或选择了不同的查找目录
- **THEN** 系统自动执行对应方向的查找并记录自动开始日志

#### Scenario: 条件缺失
- **WHEN** 当前方向缺少输入、查找目录或 RAW 格式
- **THEN** 系统不自动执行匹配，并持续展示尚未完成的步骤

#### Scenario: 先选择查找目录
- **WHEN** 用户先完成查找目录选择，之后再加入有效输入
- **THEN** 系统在输入收集完成后自动执行匹配

#### Scenario: 重新查找
- **WHEN** 当前方向已具备完整条件且用户请求重新查找
- **THEN** 系统使用当前输入、查找目录和 RAW 格式重新执行匹配
