## ADDED Requirements

### Requirement: 系统提供双向配对方向
系统 MUST支持“图片 → RAW”和“RAW → 图片”两种配对方向，并在首次进入配对工作区时默认选择“图片 → RAW”。

#### Scenario: 首次进入配对工作区
- **WHEN** 用户首次打开配对工作区且当前会话尚无方向状态
- **THEN** 系统默认显示“图片 → RAW”方向

#### Scenario: 展示当前配对方向
- **WHEN** 配对工作区处于任一方向
- **THEN** 系统在侧栏顶部持续显示当前方向，并提供可访问的轻量交换图标

### Requirement: 方向连接器位于侧栏顶部
系统 MUST将方向连接器作为两张任务卡片上方的独立控制行展示，不得将其放入任一文件卡片内部。连接器必须显示当前方向文字；切换操作可以仅显示交换图标，但必须具有无障碍标签和悬浮提示。

#### Scenario: 图片找 RAW 布局
- **WHEN** 当前方向为“图片 → RAW”
- **THEN** 上方卡片显示图片输入，下方卡片显示 RAW 查找目录，方向连接器显示“图片 → RAW”

#### Scenario: RAW 找图片布局
- **WHEN** 当前方向为“RAW → 图片”
- **THEN** 上方卡片显示 RAW 输入，下方卡片显示图片查找目录，方向连接器显示“RAW → 图片”

#### Scenario: RAW 格式控件跟随 RAW 卡片
- **WHEN** 用户切换配对方向
- **THEN** RAW 格式筛选控件必须跟随 RAW 所在卡片移动

### Requirement: 系统持续展示上下步骤关系
系统 MUST使用步骤编号、状态文字和向下连接关系表达“先提供输入文件，再选择查找目录”，但不得因第一步尚未完成而强制禁止用户提前选择查找目录。

#### Scenario: 两步均未完成
- **WHEN** 当前方向尚无有效输入且尚未选择查找目录
- **THEN** 系统将上方输入卡片标记为当前步骤，并将下方查找目录卡片显示为待完成

#### Scenario: 输入步骤完成
- **WHEN** 当前方向已加入至少一个有效输入文件
- **THEN** 系统将第一步标记为已完成，并将第二步提示为下一步

#### Scenario: 用户提前选择查找目录
- **WHEN** 用户在尚未加入输入文件时选择当前方向的查找目录
- **THEN** 系统保存该目录，同时继续提示用户完成输入步骤

### Requirement: 方向切换必须安全且状态隔离
系统 MUST为两个方向分别保存当前会话中的输入、查找目录、RAW 格式、结果、冲突确认、导出报告和日志；切换方向不得将一个方向的路径或结果解释为另一个方向的数据。

#### Scenario: 切换到尚未使用的方向
- **WHEN** 用户从已有任务的方向切换到当前会话中尚未使用的另一方向
- **THEN** 系统显示另一方向的空白任务状态，且保留原方向状态

#### Scenario: 切回已有任务的方向
- **WHEN** 用户切回当前会话中已经使用过的方向
- **THEN** 系统恢复该方向保存的输入、目录、筛选、结果、确认项、导出报告和日志

#### Scenario: 忙碌期间尝试切换方向
- **WHEN** 系统正在收集输入、执行匹配或导出文件
- **THEN** 系统禁用方向切换，并说明任务完成后才能切换

#### Scenario: 清空当前任务
- **WHEN** 用户在某一方向确认清空当前任务
- **THEN** 系统只清空当前方向的任务状态，不影响另一方向保存的任务

### Requirement: 界面语义必须随方向一致变化
系统 MUST根据当前方向同步更新页面说明、统计标签、快捷键作用对象、引导、空状态、结果列、冲突复核、查找按钮、导出按钮和运行日志中的文件角色名称。

#### Scenario: 切换为 RAW 找图片
- **WHEN** 当前方向从“图片 → RAW”切换为“RAW → 图片”
- **THEN** 系统不得继续显示“JPG 输入”“RAW 源目录”“查找 RAW”或“导出对应 RAW”等与旧方向冲突的文案

#### Scenario: 使用键盘快捷键选择文件
- **WHEN** 用户在任一方向触发输入文件或查找目录快捷键
- **THEN** 系统将快捷键操作应用到当前方向的上方输入卡片或下方查找目录卡片
