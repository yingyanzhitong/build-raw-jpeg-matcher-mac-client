# 变更日志

## 0.1.39 - 2026-07-17

- 按 `macos-app-design` 全面重构 macOS 应用壳层：将双层顶部导航收敛为统一工具栏，统一三个工作区的检查器、工具栏、状态栏与内容层级。
- 新增文件、视图和工作区原生菜单命令及对应快捷键，完善工作区、画幅、水印锚点、日志和弹窗的键盘焦点与无障碍语义。
- 优化图片/RAW 匹配、一键分离和图片水印的步骤状态、空状态主操作与渐进披露，并区分静态配对方向标签和可操作的交换按钮。
- 补充深色、高对比度、降低透明度和减少动态效果适配，以及三套工作区、最小窗口和焦点路径的前后截图与 HTML 审查报告。

## 0.1.38 - 2026-07-14

- 移除安装包发布流程中的 Gitee `pages` 分支同步，仅保留 GitHub Release 与 Gitee 发行资产同步。
- 删除独立的 Gitee Pages 同步工作流及其专用脚本，避免后续提交或发布继续写入 Gitee `pages` 分支。

## 0.1.37 - 2026-07-14

- 修复当前仓库未配置 `WECOM_WEBHOOK_KEY` 时企业微信通知被静默跳过、但 GitHub Actions 仍显示成功的问题。
- 企业微信 Secret 缺失、网络请求失败或机器人返回非零 `errcode` 时明确将通知任务标记为失败，避免发布状态误判。
- 通知成功后在 Actions 日志中输出版本确认信息，并为当前仓库补齐企业微信机器人 Secret。

## 0.1.36 - 2026-07-14

- 在 `Build Installers` 完成安装包构建、GitHub Release 与 Gitee 同步后，通过企业微信机器人发送发布成功通知。
- 通知包含版本、仓库、提交链接、触发人和 GitHub Actions 链接，并复用仓库现有的 `WECOM_WEBHOOK_KEY` Secret。
- 企业微信不可用或 Secret 缺失时仅记录 Actions 警告，不将已经成功的应用发布改判为失败。

## 0.1.35 - 2026-07-14

- 为图片水印新增全局 JPEG 导出质量设置，支持 1–100 调整并默认使用质量 100。
- 将质量设置纳入 `watermark-settings:v6` 本地持久化，旧版配置自动迁移为质量 100。
- 在 Rust 导出管线创建文件前校验质量范围，并仅对 JPEG 应用所选质量；PNG 继续保持无损编码。
- 补充配置迁移、JPEG 编码差异和 PNG 无损一致性测试，并更新简体中文 OpenSpec。

## 0.1.34 - 2026-07-14

- 将图片水印工作区的侧栏、工具栏、空状态、预览区和缩略图栏统一为图片/RAW 匹配工作区的桌面视觉与三步工作流样式。
- 为图片/RAW 匹配、一键分离和图片水印统一增加任务清空入口；分离和水印清空前会二次确认，且不会删除或改写本地文件。
- 为三个工作区统一增加右下角状态浮层，分别展示匹配、文件分类和横图/竖图/方图统计，并自动避让缩略图栏及导出结果栏。
- 补充状态映射测试以及清空、状态浮层和日志结果的交互验收，并扩展简体中文 OpenSpec 需求、设计与任务记录。

## 0.1.33 - 2026-07-14

- 以本地图片水印工作区重建旧水印模块，支持递归扫描、横图/竖图/方图独立参数、九宫格定位、旋转偏移、实时预览、批量导出、取消任务及元数据保留。
- 新增本地玻璃水印处理：不透明图片自动移除纯色背景并转换为通透玻璃效果，同时支持图片单个和平铺布局。
- 新增文字水印、本机字体选择、文字大小和通透度控制，并让旋转文字平铺按未旋转文字短边保持固定间距。
- 为图片/RAW 匹配、一键分离和图片水印统一成功/失败 Toast；失败时自动打开按 tail 定位最新记录的底部日志抽屉。
- 补充前端状态测试、Rust 图像管线测试及简体中文 OpenSpec 设计与验收文档。

## 0.1.32 - 2026-07-13

## 0.1.31 - 2026-07-13
